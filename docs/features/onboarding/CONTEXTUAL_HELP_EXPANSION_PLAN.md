## Help Content Structure

### Content Format
```markdown
## Feature Name

### Overview
Brief description of the feature

### How to Use
Step-by-step instructions

### Tips & Best Practices
Helpful tips and recommendations

### Common Issues
Frequently encountered issues and solutions

### Related Features
Links to related features

### Keyboard Shortcuts
Relevant keyboard shortcuts

### Examples
Usage examples
```

### Content Guidelines
1. **Keep it concise**: Help text should be scannable
2. **Be actionable**: Tell users what to do
3. **Use plain language**: Avoid technical jargon
4. **Provide context**: Explain why, not just what
5. **Include examples**: Show, don't just tell
6. **Link to more info**: For complex topics
